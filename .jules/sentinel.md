## 2024-05-14 - Prevent SQL Injection via Dictionary Unpacking and Assert Bypasses
**Vulnerability:** SQL injection vulnerability via parameter dictionary unpacking `**kwargs` in database abstractions (`streamrip/db.py`). The existing code attempted to validate incoming parameters with an `assert` statement against allowed schema columns before incorporating them into SQL queries.
**Learning:** `assert` statements can be globally disabled in Python when running with optimization flags (e.g., `python -O`). This removes all security and schema validations wrapped in `assert`. Additionally, accepting `**kwargs` allows bypassing Python's normal identifier character restrictions, enabling the passing of arbitrary strings (including SQL injection payloads) which would otherwise be caught if only explicit keyword arguments were used.
**Prevention:** Never use `assert` for data validation, security checks, or ensuring data integrity. Always use explicit `if not ...: raise ValueError(...)` patterns. Explicitly validate all keys originating from `**kwargs` against a strictly defined whitelist (e.g., allowed database schema columns) before using them to construct queries.
