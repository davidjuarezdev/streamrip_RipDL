## 2024-05-14 - Python `assert` Bypass Leading to SQL Injection via `**kwargs` Dictionary Unpacking
**Vulnerability:** The methods `contains` and `remove` in `streamrip/db.py` used unpacking of arbitrary dictionary keys (`**items` / `**kwargs`) to build dynamic SQL strings (`" AND ".join(f"{key}=?" for key in items.keys())`). In the `contains` method, this input validation check was protected by a standard `assert` statement (`assert all(key in allowed_keys for key in items.keys())`). The `remove` method had no validation at all.
**Learning:** `assert` statements in Python are completely ignored and skipped if the Python interpreter runs with optimization flags (e.g. `python -O`). Consequently, using `assert` for security checks, such as validating column schemas before concatenating them into an SQL query, allows attackers to bypass checks and potentially achieve SQL injection, as dictionary keys from `**kwargs` are not subject to standard Python variable identifier restrictions when supplied via unpack operations like `func(**user_input_dict)`.
**Prevention:** Always perform critical validation of input mapping to SQL elements (like columns or table names) with explicit `if` conditions that raise exceptions (e.g., `if not ...: raise ValueError`). Never use `assert` for access control, bounds-checking, or input validation. Ensure every parameter unpacking mechanism used in string formatting undergoes this explicit runtime verification.
